Lyapunov 稳定性证明应该检查什么

整理 Lyapunov 稳定性证明中最容易漏掉的几个检查点。

2026-06-02
阅读时长: 1 分钟

Lyapunov 稳定性证明看起来像一套固定模板：选一个 \(V(x)\)，证明它正定，再证明 \(\dot V(x)\le 0\) 或 \(\dot V(x)<0\)。但真正写论文或检查推导时，问题往往出在细节。

1. 平衡点是否说清楚

证明稳定性之前，必须先明确系统的平衡点是什么。对于系统

\[ \dot x = f(x) \]

如果讨论的是原点稳定性，就要确认 \(f(0)=0\)。如果系统经过坐标平移，也要说明新的误差状态如何定义。

2. \(V(x)\) 的正定性是否成立

常见说法是：

\[ V(0)=0,\quad V(x)>0,\quad x\ne 0 \]

但如果 \(V(x)\) 包含交叉项，就不能只凭直觉说它正定，而要通过矩阵正定性、Schur 补或不等式界定来证明。

3. \(\dot V\) 的条件够不够强

如果只能得到 \(\dot V\le 0\)，通常只能说明稳定性，不能直接声称渐近稳定。要得到渐近稳定，可能还需要 LaSalle 不变性原理，或者证明 \(\dot V\) 负定。

4. 结论边界要克制

不同条件对应不同结论：

  • 稳定；
  • 渐近稳定；
  • 指数稳定；
  • 一致最终有界；
  • 输入到状态稳定。

证明到哪一步，就写到哪一步。不要把“误差不发散”写成“误差收敛到零”。

5. 工程含义

在列车控制问题里，稳定性证明不是为了漂亮，而是为了说明速度误差、位置误差或跟踪误差不会因为扰动和模型误差而失控。证明的每个假设都应该能回到工程条件：限速、牵引制动约束、坡度扰动、通信延迟或执行器延时。
