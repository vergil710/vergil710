个人知识库如果只是把文件放进一个目录，很快就会变成收藏夹。真正有用的知识库应该服务具体项目，让信息能被找回、被引用、被更新。

## 1. 知识源

常见知识源包括：

- 论文 PDF；
- 读书笔记；
- 实验日志；
- 代码说明；
- 项目决策记录；
- 长对话摘要；
- 会议记录。

不是所有内容都值得入库。临时想法可以先放草稿，只有长期有用的信息才进入项目记忆。

## 2. 文档摘要

摘要不只是缩短原文，而是提取可复用信息：

- 文档目的；
- 关键结论；
- 适用场景；
- 与当前项目的关系；
- 待核验点。

## 3. 文献卡片

一篇论文的卡片应该包括：

- 问题；
- 方法；
- 实验；
- 结论；
- 局限；
- 可借鉴之处；
- 和自己工作的关系。

## 4. 项目记忆

项目记忆要记录“为什么”。比如为什么选这个模型，为什么放弃某个实验，为什么改论文结构。

这些信息最容易在几个月后丢失，却往往决定项目能不能继续推进。

知识库的目标不是保存一切，而是让重要信息在需要时能重新进入工作流。
