实验跑通一次不等于复现成功。真正的复现应该让未来的自己或别人能够重新得到同样的结果，至少知道差异来自哪里。

## 1. 环境

记录环境不是形式主义。至少包括：

- 操作系统；
- Python / MATLAB / C++ 版本；
- 关键依赖版本；
- CPU / GPU / 板端型号；
- 数据文件版本；
- 随机种子。

如果实验涉及嵌入式平台，还要记录交叉编译工具链、系统架构和运行权限。

## 2. 命令

不要只写“运行 main.py”。应该记录完整命令：

```bash
python run_experiment.py --config configs/ato_energy.yaml --seed 42
```

如果有多个阶段，也要区分数据预处理、训练、测试、绘图。

## 3. 输出

输出至少包含：

- 关键指标；
- 图表路径；
- 日志路径；
- 异常信息；
- 运行时间；
- 内存或算力开销。

## 4. 结论

实验日志最后要写判断：

- 是否复现成功；
- 与论文或预期结果是否一致；
- 差异可能来自哪里；
- 下一步需要改什么。

好的实验日志不是流水账，而是给下一次实验留下入口。
